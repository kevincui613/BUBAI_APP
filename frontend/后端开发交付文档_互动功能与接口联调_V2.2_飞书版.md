# 后端开发交付文档：互动功能与接口联调（V2.2）

## 0. 文档目的
- 给后端开发同学一份可直接执行的实现清单。
- 覆盖社交互动（好友/圈子/组队）+ AI 计划生成。
- 约束前后端字段与接口一致，减少联调反复。

## 1. 当前项目现状（重要）
- 前端已完成页面与交互：
  - 好友系统、学习圈子、搭子组队、AI 计划页。
- 前端调用模式：
  - 社交与 AI：走后端真实接口。
  - 其他核心模块（目标/日历/打卡）：暂时仍可保留 mock。
- 已实现关键产品规则：
  - 加好友必须申请并等待对方同意，不能直接成为好友。

## 2. 全局接口约定
- Base URL: `/api/v1`
- 鉴权: `Authorization: Bearer <token>`
- 响应体统一：
  - `code: number`
  - `message: string`
  - `data: any`
- 字段命名：camelCase
- 日期：`YYYY-MM-DD`
- 时间：ISO 8601

## 3. 社交功能业务规则（必须）

### 3.1 好友关系状态机
- 状态：
  - `none`：无关系
  - `pendingSent`：我已发申请
  - `pendingReceived`：对方向我发申请
  - `friend`：已互为好友
- 流转：
  - `none` -> 发申请 -> `pendingSent`
  - 对方看到 `pendingReceived`
  - 对方同意 -> 双方 `friend`
  - 对方拒绝 -> 回到 `none`
- 限制：
  - 不可添加自己
  - pending 状态不可重复发申请
  - friend 状态不可再发申请

### 3.2 推荐搭子规则（接口行为）
- 推荐列表必须排除：
  - 已是好友
  - pendingSent
  - pendingReceived
- 返回 `matchScore` 和 `reason`

### 3.3 可见性设置
- 枚举值：`public | friends | private`
- 默认：`friends`

## 4. 后端接口清单（与前端已对齐）

## 4.1 好友系统
1) `GET /social/friends/search?keyword=xxx`
- 返回 `FriendCandidate[]`
- `FriendCandidate`:
  - `id`
  - `nickname`
  - `bio`
  - `goalTag`
  - `city`
  - `streakDays`
  - `compatibility`
  - `isFriend: boolean`
  - `requestStatus: none | pendingSent | pendingReceived`

2) `POST /social/friends/requests`
- req:
  - `targetUserId: string`
- res:
  - `data: null`

3) `GET /social/friends/requests/incoming`
- 返回 `FriendRequest[]`
- `FriendRequest`:
  - `id`
  - `fromUserId`
  - `toUserId`
  - `fromNickname`
  - `fromGoalTag`
  - `status: pending | accepted | rejected`
  - `createdAt`

4) `PATCH /social/friends/requests/:requestId`
- req:
  - `action: accept | reject`
- res:
  - `data: null`

5) `GET /social/friends`
- 返回 `FriendUser[]`

6) `GET /social/friends/visibility`
- 返回 `public | friends | private`

7) `PATCH /social/friends/visibility`
- req:
  - `visibility: public | friends | private`
- 返回最新 visibility

8) `GET /social/friends/recommendations`
- 返回 `BuddyRecommendation[]`
- `BuddyRecommendation`:
  - `id`
  - `nickname`
  - `reason`
  - `matchScore`
  - `goalTag`
  - `city`

## 4.2 学习圈子
9) `GET /social/circles?category=all`
- 返回 `StudyCircle[]`
- `StudyCircle`:
  - `id`
  - `name`
  - `category`
  - `intro`
  - `memberCount`
  - `isJoined`

10) `POST /social/circles/:circleId/join`
- res:
  - `data: null`

## 4.3 搭子组队
11) `GET /social/teams/posts`
- 返回 `TeamPost[]`

12) `POST /social/teams/posts`
- req:
  - `title: string`
  - `goal: string`
  - `headcount: number` (2~20)
  - `contact: string`
- res:
  - `TeamPost`

13) `GET /social/teams/challenges`
- 返回 `TeamChallenge[]`

## 4.4 AI 计划
14) `POST /ai/plans/generate`
- req:
  - `goalTitle: string`
  - `periodDays: number` (3~30)
  - `dailyMinutes: number` (20~300)
  - `level: beginner | intermediate | advanced`
- res:
  - `AiPlanResult`
    - `summary: string`
    - `tips: string[]`
    - `tasks: { day, focus, minutes, deliverable }[]`

## 5. 错误码建议
- `code = 0` 成功
- `code = 4001` 参数错误
- `code = 4002` 重复申请/非法状态流转
- `code = 4003` 无权限操作
- `code = 4040` 资源不存在
- `code = 5000` 服务器异常

## 6. 数据库落地建议（Prisma/MySQL）
建议至少包含这些表：
- `users`
- `social_users_profile`
- `friendships`（记录发起方/接收方/status）
- `social_visibility_settings`
- `circles`
- `circle_members`
- `team_posts`
- `team_challenges`
- `team_challenge_members`
- `ai_plan_records`

关键约束：
- `friendships` unique(`userId`, `friendUserId`)
- `circle_members` unique(`circleId`, `userId`)
- 审批通过后需要建立双向好友关系（或统一以无向关系建模）

## 7. 鉴权与安全
- 生产环境不要依赖前端传昵称作为身份信息。
- 当前 `X-User-Nickname` 仅用于本地演示用户可读性。
- 正式实现必须从 JWT userId 反查用户资料。
- 大模型 key 仅后端持有，前端不可见。

## 8. AI 接入规范（Qwen）
- 环境变量：
  - `QWEN_API_KEY`
  - `QWEN_BASE_URL`
  - `QWEN_MODEL`
- 失败策略：
  - 接口超时/异常时返回 fallback 计划（保证前端可用）
- 建议：
  - 记录 prompt 摘要与结果至 `ai_plan_records`

## 9. 联调清单（后端自测 + 前端联调）

### 9.1 好友申请同意链路
1. 用户A搜索到用户B
2. A发申请成功
3. B在 incoming 列表看到申请
4. B点击同意
5. A/B 的好友列表均出现对方

### 9.2 异常用例
- A重复发送申请 -> 返回业务错误
- A给自己发申请 -> 返回业务错误
- 已是好友再次发申请 -> 返回业务错误

### 9.3 AI 用例
- 正常输入返回 `summary + tips + tasks`
- 模型异常时返回 fallback，接口仍 `code=0`

## 10. 交付优先级
- P0：好友申请同意链路 + 好友列表 + incoming + 可见性
- P1：圈子加入 + 组队招募
- P1：AI 计划生成与落库
- P2：推荐算法优化（目前可先规则版）

## 11. 前端联调开关说明
前端环境变量示例：
- `VITE_USE_MOCK=true`（保留核心模块 mock）
- `VITE_USE_SOCIAL_MOCK=false`（社交走后端）
- `VITE_USE_AI_MOCK=false`（AI 走后端）
- `VITE_API_BASE_URL=http://localhost:3000/api/v1`
