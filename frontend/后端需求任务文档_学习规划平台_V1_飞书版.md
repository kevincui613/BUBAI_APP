# 个性化学习规划与社交监督平台 V1

## 1. 文档目的
- 本文用于后端开发同学按前端现状快速落地 API 与数据库。
- 当前前端已完成 V1 页面和交互，且已用 Mock 数据打通流程。
- 本文只覆盖 V1 最小可运行范围，不包含社交功能，不包含 V1.5/V2。

## 2. V1 范围（必须）
- 注册/登录
- 目标管理（创建、列表）
- 计划日历（月视图 + 日详情）
- 任务完成打卡
- 统计展示（连续打卡天数、本周完成率）

## 3. 前端已完成内容
- 路由页面
  - /login 登录
  - /register 注册
  - /dashboard 概览
  - /goals 目标管理
  - /calendar 计划日历
- 已实现 API 封装
  - auth/register/login/me
  - goals/create/list
  - calendar/month/day
  - tasks/checkin
  - stats/overview
- 当前默认使用本地 Mock（方便演示）
  - 环境变量：VITE_USE_MOCK=true
  - 后端联调时改为：VITE_USE_MOCK=false

## 4. 数据字段统一规范
- API 字段统一使用 camelCase
- 日期字段统一 YYYY-MM-DD
- 时间字段统一 ISO 8601 字符串
- 主键返回 string（避免前端 number 精度问题）

## 5. 数据库表建议（MySQL + Prisma）

### 5.1 users
- id: bigint PK
- email: varchar(191) unique
- passwordHash: varchar(255)
- nickname: varchar(50)
- timezone: varchar(50) default Asia/Shanghai
- createdAt: datetime(3)
- updatedAt: datetime(3)

### 5.2 goals
- id: bigint PK
- userId: bigint FK -> users.id
- title: varchar(100)
- description: varchar(500) nullable
- targetTotal: int
- unit: varchar(20)
- startDate: date
- endDate: date
- status: enum(active, completed, archived)
- createdAt: datetime(3)
- updatedAt: datetime(3)

### 5.3 dailyTasks
- id: bigint PK
- userId: bigint FK -> users.id
- goalId: bigint FK -> goals.id
- taskDate: date
- title: varchar(120)
- plannedAmount: int
- completedAmount: int default 0
- isCompleted: boolean default false
- completedAt: datetime(3) nullable
- note: varchar(255) nullable
- createdAt: datetime(3)
- updatedAt: datetime(3)
- unique(goalId, taskDate)

## 6. API 需求清单（必须按此命名）

### 6.1 Auth
1) POST /api/v1/auth/register
- req
  - email: string
  - password: string
  - nickname: string
- res.data
  - user: { id, email, nickname }
  - accessToken: string

2) POST /api/v1/auth/login
- req
  - email: string
  - password: string
- res.data
  - user: { id, email, nickname }
  - accessToken: string

3) GET /api/v1/auth/me
- header
  - Authorization: Bearer <token>
- res.data
  - { id, email, nickname }

### 6.2 Goals
4) POST /api/v1/goals
- req
  - title: string
  - description?: string
  - targetTotal: number
  - unit: string
  - startDate: string
  - endDate: string
- 行为
  - 创建目标后自动按周期生成 dailyTasks
- res.data
  - goal 对象

5) GET /api/v1/goals?status=active
- res.data
  - goal[]

### 6.3 Calendar + Tasks
6) GET /api/v1/calendar/month?year=2026&month=4
- res.data
  - [{ date, totalTasks, completedTasks, completionRate }]

7) GET /api/v1/calendar/day?date=2026-04-13
- res.data
  - [{ id, goalId, taskDate, title, plannedAmount, completedAmount, isCompleted, note }]

8) PATCH /api/v1/tasks/:taskId/checkin
- req
  - isCompleted: boolean
  - completedAmount?: number
  - note?: string
- res.data
  - task 对象

### 6.4 Stats
9) GET /api/v1/stats/overview
- res.data
  - currentStreakDays: number
  - weeklyCompletionRate: number
  - todayCompletedTasks: number
  - todayTotalTasks: number

## 7. 统一响应体
- 成功
  - code: 0
  - message: success
  - data: any
- 失败
  - code: 非 0
  - message: 错误信息
  - data: null

## 8. 核心业务规则（必须实现）
- 所有业务数据按当前登录用户隔离。
- 创建目标时，系统自动生成从 startDate 到 endDate 的每日任务。
- 任务打卡规则
  - isCompleted=true 时，completedAmount 默认等于 plannedAmount（若前端未传）
  - isCompleted=false 时，completedAmount 重置为 0
- 连续打卡天数
  - 从今天向前按天计算，只要当天任务全部完成才算连续
  - 某天无任务则中断
- 本周完成率
  - 周一到周日范围内，已完成任务数 / 总任务数

## 9. 联调要求
- 跨域：允许前端开发域名
  - http://localhost:5173
- 鉴权：JWT Bearer
- 时间基准：优先使用 Asia/Shanghai
- 返回字段与命名必须严格对齐第 6 节

## 10. 验收标准（后端）
- 能通过真实接口跑通以下完整链路：
  - 注册 -> 登录 -> 创建目标 -> 月历查看 -> 日详情查看 -> 打卡 -> 概览统计变化
- 前端切换 VITE_USE_MOCK=false 后无需改动页面代码即可联调成功。
- 所有接口具备基础参数校验与错误返回。
